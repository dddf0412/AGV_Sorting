# 嵌赛 ST平台 智能分拣AGV 项目

## 项目简介

基于 STM32 双 MCU 主从架构的智能分拣 AGV（自动导引车）系统。通过语音指令或屏幕触控下达分拣任务，系统进行视觉识别、NFC 身份认证、电机运动控制完成自动分拣流程。

## 工程目录

| 目录 | 说明 |
|------|------|
| `AGV_H753_Master/` | 主控板工程 (STM32H753ZITX)，CubeIDE 项目 |
| `G474_AGV_Slave/` | 从板工程 (STM32G474RETX)，CubeIDE 项目 |
| `docs/` | 项目技术文档 |
| `tools/` | 工具脚本 (Python) |

### 代码架构

```
Core/Src/main.c          ← 仅外设初始化 + App_Init() + while(1) App_Run()
Core/Src/app_h753.c      ← H753 应用层：主循环逻辑、模式切换、子系统调度
Core/Src/app_g474.c      ← G474 应用层：主循环逻辑、模式切换、子系统调度
Core/Src/h753_can.c      ← H753 CAN 协议层 (发送/接收/回调)
Core/Src/g474_can.c      ← G474 CAN 协议层 (发送/接收/回调)
Core/Src/nfc.c           ← NFC MFRC-522 驱动 + 状态机 + 白名单
Core/Src/microphone.c    ← MP34DT01 PDM 麦克风驱动 (DFSDM DMA + 实时流)
Core/Src/screen.c        ← 淘晶驰 T1 串口屏驱动 (USART1, 9600)
Core/Src/camera.c        ← OV2640 摄像头驱动 + DCMI JPEG 流传输 (SCCB 软件 I2C)
Core/Src/sai.c           ← SAI1 音频接口 (CubeMX 生成)
Core/Src/motor_control.c ← G474 电机/舵机/编码器/传感器 API + 1kHz 控制循环
Core/Src/agv_g474.c      ← G474 状态管理 (电机锁定/解锁, 状态结构体)
```

### 运行模式切换

两个应用层头文件 `app_h753.h` / `app_g474.h` 中定义：

```c
#define APP_MODE  0  // 0=模拟模式 (默认), 1=真实模式
```

- **模拟模式 (0)**：生成假传感器/编码器数据，CAN 交互只 log 不驱动硬件
- **真实模式 (1)**：启用全部硬件驱动，执行真实控制逻辑

切换模式只需改此数字重新编译，无需修改 main.c。

## 系统架构

```
[用户层]
  语音(MIC)  NFC刷卡  串口屏
     │         │        │
     ▼         ▼        ▼
┌──────────────────────────────┐
│  H753 Master (高算力AI决策)   │
│  STM32H753ZITX, 480MHz       │
│  - 语音AI推理                 │
│  - 视觉图像处理 (摄像头)       │
│  - NFC 身份认证               │
│  - 数字孪生显示 (串口屏)       │
│  - CAN 总线调度               │
└─────────────┬────────────────┘
              │ FDCAN 1Mbps
              │ 0x20x: 指令下发  |  0x301: 状态上报  |  0x2FF: 心跳
              │
┌─────────────┴────────────────┐
│  G474 Slave (硬实时运动控制)   │
│  STM32G474RET6, 170MHz       │
│  - 1kHz 控制循环              │
│  - 电机控制 (DC/步进/舵机)     │
│  - 编码器反馈                 │
│  - 循迹传感器采集              │
│  - 安全保护                   │
└──────────────────────────────┘
```

**职责边界：** H753 不直接控制任何电机/传感器，所有执行动作通过 CAN 指令委托 G474 完成。G474 不处理业务逻辑，只解析 CAN 指令执行硬件操作并回传数据。

## CAN 通信协议 (FDCAN, 1Mbps, Classic CAN)

### H753 → G474 (命令帧)

| CAN ID | 指令 | Data[0] | Data[1] | Data[2-7] |
|--------|------|---------|---------|-----------|
| **0x201** | AGV 运动指令 | 速度 (0-255) | 方向 (0=正转, 1=反转) | 预留 |
| **0x202** | 传送带指令 | 启停 (0=停, 1=启) | 转速 (0-255) | 方向(Data[2]) + 预留 |
| **0x203** | 舵机云台指令 | 俯仰角度 (0-180) | 偏航角度 (0-180) | 预留 |
| **0x2FF** | 心跳帧 | 0xAA | 预留 | 预留 |

### G474 → H753 (状态帧)

| CAN ID | 指令 | DLC | 数据布局 |
|--------|------|-----|----------|
| **0x301** | 状态上报 | 8 | [0]直流速度(int8) [1]步进速度(uint8) [2-3]编码器计数(int16, LE) [4]左传感器灰度 [5]右传感器灰度 [6]系统状态字 [7]尾帧=0xAA |

**系统状态字 (Data[6]) 位定义：**
- Bit 0: 脱轨标志 (双传感器均检测白色)
- Bit 1: 通信超时标志

## 安全机制

| 场景 | 检测方 | 执行动作 |
|------|--------|----------|
| AGV 脱轨 | G474 | 双传感器均白色 → 立即停车 → 上报 H753 |
| CAN 超时 (G474 侧) | G474 | 500ms 未收到心跳 → 停止所有电机 |
| CAN 超时 (H753 侧) | H753 | 500ms 未收到状态帧 → 屏幕 + 语音报警 |
| 电机过流 | 硬件 | DRV8825/L298N 内置保护 |

## AI 管线架构 (Design for AI)

```
┌─────────────────────────────────────────────────────────┐
│                    用户交互层                             │
│  语音输入 (MIC)    NFC 刷卡      串口屏 (触控+显示)        │
│      │                │               │                  │
│      ▼                ▼               ▼                  │
│  ┌───────────────────────────────────────────────────┐  │
│  │            H753 Master — AI 推理决策                │  │
│  │  STM32H753ZITX, 480MHz Cortex-M7 + DSP             │  │
│  │                                                    │  │
│  │  ① KWS 关键词识别 (TFLite Micro, 5-8 词)  ~40%CPU  │  │
│  │  ② 视觉颜色/形状识别                 ~30%CPU       │  │
│  │  ③ AI 决策状态机                     ~5%CPU        │  │
│  │  ④ 语音反馈 (WM8960 SD 卡 WAV 流播)  DMA           │  │
│  │  ⑤ CAN 指令下发 (FDCAN 1Mbps)        <1%CPU        │  │
│  │  ⑥ FreeRTOS 任务调度 (后期待引入)                   │  │
│  │                                                    │  │
│  │  ~75% CPU 利用率, DMA 搬运声音/图像/音频             │  │
│  └────────────────────┬──────────────────────────────┘  │
│                       │ CAN 1Mbps, 8B 指令, <1ms 时延    │
│  ┌────────────────────┴──────────────────────────────┐  │
│  │            G474 Slave — 硬实时运动控制               │  │
│  │  STM32G474RET6, 170MHz Cortex-M4                   │  │
│  │                                                    │  │
│  │  ① 1kHz 定时器中断 (TIM6, 硬件触发, 确定性)  <1%    │  │
│  │  ② PID 直流速度闭环                    ~2%CPU      │  │
│  │  ③ 步进梯形/S 曲线加减速                ~1%CPU      │  │
│  │  ④ 编码器航位推算 + 速度/加速度计算     ~2%CPU      │  │
│  │  ⑤ 循迹纠偏 + 脱轨预测                  ~1%CPU      │  │
│  │  ⑥ CAN 状态上报 (FDCAN 1Mbps)           <1%CPU      │  │
│  │                                                    │  │
│  │  硬实时控制回路, 不参与 AI 推理, 不受 AI 抖动影响     │  │
│  │  ~8% CPU 利用率, 充足安全余量确保运动确定性           │  │
│  └───────────────────────────────────────────────────┘  │
│                          │                              │
│                          ▼                              │
│              电机/舵机/编码器/循迹传感器                    │
└─────────────────────────────────────────────────────────┘
```

**架构设计理念:**
- **异构双核**: Cortex-M7 做 AI, Cortex-M4 做硬实时 — H753 不控电机, G474 不碰 AI
- **职责隔离**: 1kHz 控制回路用 TIM6 硬件触发, 不受 H753 AI 推理负载波动影响
- **确定性通信**: CAN 1Mbps 8 字节指令 <1ms 时延, 心跳 100ms, 超时 500ms 自动停车
- **FreeRTOS 计划**: H753 侧任务化 (AI 推理 / NUC / 屏幕 / CAN 调度线程), G474 侧保持裸机 (硬实时确定性)

## 调试方式

- **H753 调试 + 音频流**: USART3, PD8(TX)/PD9(RX), 3Mbps
- **G474 调试串口**: LPUART1, PA2(TX)/PA3(RX), 115200-8N1
- **麦克风实时播放**: `python tools/mic_stream.py COM3`
- **摄像头实时预览**: `python tools/camera_viewer.py COM3`
- **H753 NFC 命令接口**: 串口助手连 USART3, 命令格式 `NFC:IDLE` / `NFC:ADD` / `NFC:VERIFY` / `NFC:LIST` / `NFC:CLEAR`
- **CAN 监听**: USB-CAN 分析仪 → 上位机
- **printf** 已在两个项目重定向
