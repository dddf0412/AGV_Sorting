# CAN/FDCAN 双板通信技术规格书

**适用平台**: STM32H753ZI (Master) + STM32G474RE (Slave)
**版本**: V1.0
**日期**: 2026-05-29

---

## 1. 模块概述

本模块基于 STM32 内置 FDCAN 外设，为嵌赛ST平台双 MCU 架构提供实时通信总线。H753 主控与 G474 从站通过 1Mbps CAN 2.0 总线交换指令和状态数据。

### 模块设计目标

| 目标 | 说明 |
|:---|:---|
| 松耦合 | 两板仅通过 CAN 报文交互，各自内部实现完全独立 |
| 硬实时 | G474 以 100Hz 上报状态，H753 以 10Hz 下发心跳 |
| 安全优先 | 双端均有心跳超时检测（500ms），脱轨/断连自动停车 |
| 易调试 | 所有 CAN 帧收发均通过串口打印，带统一前缀 |
| 协议清晰 | 5 种 CAN ID 覆盖所有交互场景，扩展预留充分 |

## 2. 硬件连接

### 2.1 H753 主控端

| 信号名 | 引脚 | 类型 | 连接对象 |
|:---|:---|:---|:---|
| FDCAN1_TX | PD1 | AF9 复用 | SN65HVD230 TXD |
| FDCAN1_RX | PD0 | AF9 复用 | SN65HVD230 RXD |
| 3.3V | — | 电源 | SN65HVD230 VCC (经 AMS1117-3.3) |
| GND | — | 地 | SN65HVD230 GND (单点共地) |

### 2.2 G474 从站端

| 信号名 | 引脚 | 类型 | 连接对象 |
|:---|:---|:---|:---|
| FDCAN1_TX | PA12 | AF 复用 | SN65HVD230 TXD |
| FDCAN1_RX | PA11 | AF 复用 | SN65HVD230 RXD |
| 3.3V | — | 电源 | SN65HVD230 VCC |
| GND | — | 地 | SN65HVD230 GND (单点共地) |

### 2.3 硬件注意事项

- CAN 收发器选用 **SN65HVD230**，供电 3.3V（不可接 5V）
- 总线两端各并联 **120Ω 终端电阻**，缺一不可
- 所有模块单点共地，避免地电位差引入共模干扰
- CAN 总线双绞线布线，远离电机 PWM 等大电流走线
- 若通信距离超过 2m，建议降低波特率至 500kbps

## 3. 软件架构

### 3.1 分层结构

| 层级 | H753 组成 | G474 组成 | 职责 |
|:---|:---|:---|:---|
| 硬件抽象层 | HAL_FDCAN | HAL_FDCAN | CubeMX 生成的 FDCAN 外设驱动 |
| 协议层 | h753_can.c | g474_can.c | CAN 帧封装/解析、过滤器配置、中断回调 |
| 应用层 | app_h753.c | app_g474.c | 心跳调度、指令分发、状态处理、模式切换 |

### 3.2 数据流向

```
H753 Master → h753_can.c → HAL_FDCAN → SN65HVD230 → CAN Bus → SN65HVD230 → HAL_FDCAN → g474_can.c → G474 Slave

H753 → G474: 0x201(AGV指令), 0x202(传送带指令), 0x203(舵机指令), 0x2FF(心跳)
G474 → H753: 0x301(状态上报)
```

### 3.3 通信模式

- **全双工**: FDCAN 硬件自动仲裁，无需软件调度
- **发送策略**: H753 100ms 周期发心跳 + 事件驱动发指令；G474 100ms 周期发状态帧
- **接收策略**: 两端均使用 FDCAN RX FIFO0 中断，ISR 中仅做数据转存和标志置位

## 4. CAN 总线物理层参数

| 参数 | H753 值 | G474 值 | 说明 |
|:---|:---|:---|:---|
| 波特率 | 1 Mbps | 1 Mbps | 经典 CAN (非 FD) |
| APB1 时钟 | 120 MHz | 170 MHz | FDCAN 外设时钟源 |
| Nominal Prescaler | 12 | 17 | 分频系数 |
| TimeSeg1 (BS1) | 7 | 7 | 位时间段1 |
| TimeSeg2 (BS2) | 2 | 2 | 位时间段2 |
| SJW | 1 | 1 | 同步跳转宽度 |
| 时间量子 (Tq) | 100 ns | 100 ns | 1/(APB1_CLK/Prescaler) |
| 采样点 | 80% | 80% | (1+Seg1)/(1+Seg1+Seg2) |

- **帧格式**: Classic CAN 2.0A，标准帧 (11-bit ID)，数据长度固定 8 字节 (DLC=8)
- **滤波器**: 两端均配置为接受全部标准帧 (Mask=0x7FF, Filter=0x000)，软件层分发

## 5. 报文协议定义

### 5.1 CAN ID 分配表

| CAN ID | 方向 | 指令名称 | 触发方式 | 说明 |
|:---|:---|:---|:---|:---|
| 0x201 | H753→G474 | AGV 运动指令 | 事件驱动 | 控制 AGV 底盘直流电机 |
| 0x202 | H753→G474 | 传送带指令 | 事件驱动 | 控制传送带步进电机 |
| 0x203 | H753→G474 | 舵机指令 | 事件驱动 | 控制二维云台角度 |
| 0x2FF | H753→G474 | 心跳帧 | 100ms 周期 | 主站生命信号 |
| 0x301 | G474→H753 | 状态上报 | 100ms 周期 | 电机/传感器/编码器状态 |

### 5.2 H753 → G474 命令帧

**0x201 — AGV 运动指令**

| 字节 | 字段名 | 类型 | 取值范围 | 说明 |
|:---|:---|:---|:---|:---|
| Data[0] | 速度 | uint8_t | 0~255 | PWM 占空比映射 |
| Data[1] | 方向 | uint8_t | 0/1 | 0=正转, 1=反转 |
| Data[2-7] | 预留 | — | — | 加速度、距离等 |

**0x202 — 传送带指令**

| 字节 | 字段名 | 类型 | 取值范围 | 说明 |
|:---|:---|:---|:---|:---|
| Data[0] | 启停 | uint8_t | 0/1 | 0=停止, 1=启动 |
| Data[1] | 转速 | uint8_t | 0~255 | 步进电机目标转速 |
| Data[2] | 方向 | uint8_t | 0/1 | 0=正转, 1=反转 |
| Data[3-7] | 预留 | — | — | 运行时长、位置等 |

**0x203 — 舵机指令**

| 字节 | 字段名 | 类型 | 取值范围 | 说明 |
|:---|:---|:---|:---|:---|
| Data[0] | 俯仰角度 | uint8_t | 0~180 | 舵机 1 目标角度 |
| Data[1] | 偏航角度 | uint8_t | 0~180 | 舵机 2 目标角度 |
| Data[2-7] | 预留 | — | — | — |

**0x2FF — 心跳帧**

| 字节 | 字段名 | 类型 | 取值 | 说明 |
|:---|:---|:---|:---|:---|
| Data[0] | 标记 | uint8_t | 0xAA | 心跳标识 |
| Data[1-7] | 预留 | — | — | — |

### 5.3 G474 → H753 状态帧 (0x301)

| 字节 | 字段名 | 类型 | 取值范围 | 说明 |
|:---|:---|:---|:---|:---|
| Data[0] | 直流速度 | int8_t | -128~+127 | 有符号速度值 |
| Data[1] | 步进速度 | uint8_t | 0~255 | 步进电机实际速度 |
| Data[2-3] | 编码器 | int16_t | -32768~+32767 | 16位编码器 (小端) |
| Data[4] | 左传感器 | uint8_t | 0~255 | TCRT5000 (12bit ADC >> 4) |
| Data[5] | 右传感器 | uint8_t | 0~255 | TCRT5000 (12bit ADC >> 4) |
| Data[6] | 系统状态 | uint8_t | 按位 | 见下表 |
| Data[7] | 尾帧 | uint8_t | 0xAA | 帧尾校验标记 |

**Data[6] 系统状态字:**

| 位 | 名称 | 置位条件 | 说明 |
|:---|:---|:---|:---|
| Bit 0 | 脱轨标志 | 左右传感器同时白色 | G474 已自动停车 |
| Bit 1 | 通信超时 | 500ms 未收到心跳 | 全部电机进入安全态 |
| Bit 2 | 电机过流 | 预留 | DRV8825/L298N 硬件保护 |
| Bit 3-7 | 预留 | — | 电池电量、温度等 |

### 5.4 协议版本兼容规则

- 向后兼容：新增字段使用预留字节，旧接收端忽略未定义字段
- 帧识别：Data[7]=0xAA 作为状态帧有效标志
- 心跳识别：同时检查 ID=0x2FF 且 Data[0]=0xAA

## 6. API 接口

### 6.1 H753 端 (h753_can.h)

| 函数原型 | 调用时机 | 返回值 | 说明 |
|:---|:---|:---|:---|
| `void H753_CAN_Init(void)` | main() 初始化 | 无 | 配置滤波器，启动 FDCAN |
| `uint8_t H753_CAN_SendHeartbeat(void)` | 100ms 周期 | 0/1 | 发送心跳帧 0x2FF |

| `uint8_t H753_CAN_SendConveyorCmd(uint8_t run, uint8_t speed, uint8_t dir)` | 决策层触发 | 0/1 | 发送传送带指令 0x202 |
| `uint8_t H753_CAN_SendRaw(uint32_t id, uint8_t *data, uint8_t len)` | 内部/扩展 | 0/1 | 通用 CAN 发送 |
| `uint8_t H753_CAN_IsStatusFrameValid(void)` | 主循环轮询 | 1/0 | 检查新状态帧 |
| `void H753_CAN_GetStatus(H753_StatusFrame_t *out)` | 收到状态帧后 | 无 | 解析最新状态 |
| `void H753_CAN_ClearStatusFlag(void)` | 处理完状态帧后 | 无 | 清除标志 |

### 6.2 G474 端 (g474_can.h)

| 函数原型 | 调用时机 | 返回值 | 说明 |
|:---|:---|:---|:---|
| `void G474_CAN_Init(void)` | main() 初始化 | 无 | 配置滤波器，启动 FDCAN |
| `uint8_t G474_CAN_SendStatus(const G474_StatusFrame_t *status)` | 100ms 周期 | 0/1 | 发送状态帧 0x301 |
| `uint8_t G474_CAN_SendRaw(uint32_t id, uint8_t *data, uint8_t len)` | 内部/扩展 | 0/1 | 通用 CAN 发送 |
| `uint32_t G474_CAN_GetLastRxID(void)` | 主循环轮询 | 非0=有新指令 | 获取最新 CAN ID |

### 6.3 约束

**main.c 禁止直接调用 can 层函数**，所有 CAN 操作统一通过 `app_*.c` 应用层调度。

## 7. 典型通信时序

### 7.1 正常分拣流程

H753 以 100ms 周期发送心跳。语音/视觉识别触发决策后，通过 CAN 下发指令。G474 以 100ms 周期上报状态帧。

典型序列: `语音"分拣红色物品"` → `图像识别` → `舵机0x203对准` → `传送带0x202启动` → `物品到位停止`

### 7.2 安全保护

- **心跳超时**: G474 500ms 无心跳 → 停止全部电机 → 状态帧 Bit1=1 上报
- **脱轨保护**: 双传感器白色 → 立即停车 → 状态帧 Bit0=1 → H753 告警
- **NFC 授权**: 刷卡通过 → 下发解锁指令 → G474 清除 motor_locked 标志

## 8. 日志输出规范

| 前缀 | 板端 | 含义 | 示例 |
|:---|:---|:---|:---|
| [H753] | Master | 主站 CAN 事件 | `[H753] Send heartbeat ID=0x2FF` |
| [G474] | Slave | 从站 CAN 事件 | `[G474] Send status ID=0x301` |
| [CAN] | 双端 | CAN 驱动层事件 | `[CAN] Sent ID=0x201` |
| [Sim decision] | H753 | 模拟模式决策 | `[Sim decision] Send AGV cmd` |
| [Sim exec] | G474 | 模拟模式执行 | `[Sim exec] AGV cmd` |
| [G474 Safety] | G474 | 安全事件 | `[G474 Safety] No heartbeat >500ms!` |

- **模拟模式 (APP_MODE=0)**: CAN 指令仅打印不执行硬件
- **真实模式 (APP_MODE=1)**: CAN 指令驱动真实电机
- H753 日志端口: USART3 115200-8N1
- G474 日志端口: LPUART1 115200-8N1

## 10. 与外部模块联动

### 10.1 H753 端

| 联动模块 | 触发条件 | CAN 动作 |
|:---|:---|:---|
| NFC 认证通过 | NFC_Task() 返回 GRANTED | 发送 0x201/0x202 解锁指令 |
| NFC 认证失败 | NFC_Task() 返回 DENIED | 不发送解锁指令 |
| 语音识别 | 关键词识别成功 | 发送对应 CAN 指令 |
| 视觉识别 | 颜色/位置识别完成 | 发送 0x203 + 0x202 |
| 串口屏 | 用户触控指令 | 发送对应 CAN 指令 |
| 安全监控 | 状态帧 Bit0/Bit1 置位 | 触发音频 + 屏幕报警 |

### 10.2 G474 端

| 联动模块 | 触发条件 | CAN 动作 |
|:---|:---|:---|
| 1kHz 控制循环 | 每 10 次循环 | 发送 0x301 状态帧 |
| 脱轨检测 | 双传感器白色 | 状态帧 Bit0=1 |
| 心跳超时 | 500ms 无心跳 | 状态帧 Bit1=1 |
| CAN 指令接收 | 收到 0x201/202/203 | 执行 motor_control 函数 |

## 11. 故障排查

| 现象 | 可能原因 | 排查方法 |
|:---|:---|:---|
| 编译报错 "FDCAN undefined" | CubeIDE 索引器未刷新 | 右键项目 → Index → Rebuild |
| 上电后无 CAN 日志 | FDCAN 未初始化 | 检查 MX_FDCAN1_Init 调用顺序 |
| 心跳发送但 G474 收不到 | 总线断开、终端电阻缺失 | 示波器测 CAN_H/CAN_L 波形 |
| 状态帧收到但数据异常 | Data[7]≠0xAA | 检查 G474 数据打包逻辑 |
| G474 不断打印 "No heartbeat" | H753 心跳未发送 | 检查 HAL_GetTick 计数 |
| CAN 发送失败 | TX FIFO 满 | 检查 FDCAN Error 计数器 |
| 模拟正常但真实模式无反应 | motor_locked=true | 确认 NFC 认证已通过 |

### 调试技巧

- 最小化测试：先用 USB-CAN 分析仪监听 H753 发送端
- 环回测试：H753 CAN_TX 与 CAN_RX 短接验证自收自发
- 波特率验证：示波器测量单个位宽是否为 1μs (1Mbps)
- 帧尾校验：0x301 帧 Data[7] 必须为 0xAA
