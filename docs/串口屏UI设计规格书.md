# 串口屏 UI 设计规格书

项目: 嵌赛 ST平台 智能分拣AGV
屏幕: 淘晶驰 T1 4.3寸 480×272 电容触摸屏
通信: USART1, 9600bps (出厂默认)
工程名: `AGV_Sorting.HMI`

---

## 一、页面总览 (6页) 与 认证流程

### 页面结构

| 页码 | 页面名 | 用途 | 访问条件 |
|------|--------|------|----------|
| page 0 | `lock` | 安全锁定页 — 认证入口 | 上电默认，始终可访问 |
| page 1 | `main` | 数字孪生系统总览 | 认证通过后跳转 |
| page 2 | `motor` | 电机与传感器 (G474 从站数据) | 从 main 导航 |
| page 3 | `can_ctrl` | CAN 指令下发 (控制面板) | 从 main 导航，需已解锁 |
| page 4 | `auth` | NFC 白名单管理 | 从 main 导航 |
| page 5 | `system` | 系统信息与调试 | 从 main 导航 |

### 认证流程

```
上电 ──► page 0 (lock)
           │  "系统已锁定"
           │  "请刷卡或语音认证"
           │
    ┌──────┴──────┐
    ▼              ▼
 NFC刷卡        语音声纹
    │              │
    └──────┬──────┘
           ▼
       H753 验证
           │
    ┌──────┴──────┐
    ▼              ▼
  授权通过       授权失败
    │              │
    ▼              ▼
 page 1 (main)   page 0 停留
 解锁G474电机    "未授权卡片"
                 红色警告
```

### 锁定/解锁机制

- **上电默认 page 0** (`lock`)，所有电机锁定，CAN 控制页不可访问
- **认证通过** → MCU 发 `page main` + `AGV_UnlockMotors()`
- **手动锁定** → main/motor 页底部"锁定"按钮 → MCU 收到后发 `page lock` + `AGV_LockMotors()`
- **安全超时** (可选) → 60s 无操作 → 自动退回 lock 页

## 二、Page 0 — 安全锁定页 `lock`

**定位:** 上电默认页，系统入口。未认证前所有功能锁定，相当于手机锁屏。

**布局:** 居中显示锁图标/文字，下半部分状态提示。

### 控件清单

| 控件名 | 类型 | 用途 | MCU推送 |
|--------|------|------|---------|
| `t0` | 文本 | 页标题 "嵌赛智能分拣AGV" | 静态 |
| `t1` | 文本 | 锁定状态: "系统已锁定" / "认证通过" | `Screen_SetText("t1","%s",msg)` |
| `t2` | 文本 | 提示:"请刷卡 或 语音认证" | 静态/动态 |
| `t3` | 文本 | 认证结果: 空白/未授权(红)/通过(绿) | `Screen_SetText("t3","%s",res)` |
| `t4` | 文本 | CAN 连接状态 | G474已连接/G474断开 |

### 布局

```
┌──────────────────────────────────────┐
│                                      │
│          t0: 嵌赛智能分拣AGV          │
│                                      │
│            🔒 (图标/图片)             │
│          t1: 系统已锁定               │
│          t2: 请刷卡 或 语音认证        │
│          t3: (认证结果动态)            │
│                                      │
│          t4: G474 未连接              │
│                                      │
└──────────────────────────────────────┘
```

### 交互逻辑

- **上电默认**: 显示锁定界面，所有控件灰色
- **NFC 刷卡 + 授权通过**: MCU 发 `Screen_SetText("t3","✓ 授权通过")` → 延迟 1s → MCU 发 `page main`
- **NFC 刷卡 + 授权失败**: MCU 发 `Screen_SetText("t3","✗ 未授权卡片")` + 红色 → 停留 lock 页
- **语音认证通过**: 同上流程
- **无触摸事件**: lock 页不需要按钮，由 MCU 控制跳转

## 三、Page 1 — 主界面 `main`

**定位:** 认证通过后的首页，数字孪生总览。一眼看清 AGV 完整运行状态。

**访问条件:** 仅从 lock 页认证通过后由 MCU 跳转进入。

**布局:** 上 2/3 为状态卡片区, 下 1/3 为导航按钮栏 + 锁定按钮。

### 控件清单

| 控件名 | 类型 | 用途 | 数据来源 / 说明 |
|--------|------|------|-----------------|
| `t0` | 文本 | 页标题 "智能分拣AGV 数字孪生" | 静态 |
| `t_mode` | 文本 | 当前系统模式 | "模拟模式" / "真实运行" |
| `n_sort` | 数字 | **分拣计数** (累计分拣物品数) | MCU 累加推送 |
| `n0` | 数字 | AGV 直流电机速度 | CAN 0x301 `dc_speed` |
| `n1` | 数字 | 步进电机速度 | CAN 0x301 `step_speed` |
| `n2` | 数字 | 编码器计数值 | CAN 0x301 `encoder_cnt` |
| `t_sensor` | 文本 | 循迹传感器状态 | "正常循迹" / "脱轨!" |
| `t_conn` | 文本 | CAN 连接状态 | "G474已连接" / "G474断开" |
| `t_auth` | 文本 | NFC 认证状态 | "已认证" / "未认证" |
| `b0` | 按钮 | 跳转 motor 页 | 触摸 → `page motor` |
| `b1` | 按钮 | 跳转 can_ctrl 页 | 触摸 → `page can_ctrl` |
| `b2` | 按钮 | 跳转 auth 页 | 触摸 → `page auth` |
| `b3` | 按钮 | 跳转 system 页 | 触摸 → `page system` |
| `b_lock` | 按钮 | **锁定系统** (红色) | MCU 回调: 锁电机 + `page lock` |

### 布局示意

```
┌──────────────────────────────────────┐
│  t0: 智能分拣AGV 数字孪生   t_mode   │
├────────────────┬─────────────────────┤
│  AGV 底盘      │  传送带             │
│  速度: n0      │  速度: n1          │
│  编码器: n2    │                   │
├────────────────┼─────────────────────┤
│  分拣计数       │  系统状态           │
│  n_sort        │  t_sensor          │
│                │  t_conn            │
│                │  t_auth            │
├────────────────┴─────────────────────┤
│ [motor] [can] [auth] [sys]  [🔒锁定] │
└──────────────────────────────────────┘
```

### 触摸事件

| 按钮 | 事件 | 执行代码 |
|------|------|----------|
| `b0` | 按下 | `page motor` |
| `b1` | 按下 | `page can_ctrl` |
| `b2` | 按下 | `page auth` |
| `b3` | 按下 | `page system` |
| `b_lock` | 按下 | MCU 回调: 锁定电机 + `page lock` |

## 四、Page 2 — 电机与传感器 `motor`

**定位:** 详细展示 G474 从站的所有传感器和执行器状态。数据来源: CAN 0x301 状态帧。

**布局:** 左半屏电机数据, 右半屏传感器数据。

### 控件清单

| 控件名 | 类型 | 数据来源 | 用途 |
|--------|------|----------|------|
| `t0` | 文本 | 静态 | 页标题 "电机与传感器 (G474)" |
| `n0` | 数字 | `status.dc_speed` | 直流电机速度 (PWM 占空比) |
| `n1` | 数字 | `status.step_speed` | 步进电机速度 |
| `n2` | 数字 | `status.encoder_cnt` | 编码器 16 位计数值 |
| `n3` | 数字 | `status.sensor_left` | 左 TCRT5000 灰度值 (0-255) |
| `n4` | 数字 | `status.sensor_right` | 右 TCRT5000 灰度值 (0-255) |
| `t1` | 文本 | 动态 | 脱轨状态: "正常" / "脱轨! 已停车" |
| `t2` | 文本 | 动态 | 电机锁定状态: "已锁定" / "已解锁" |
| `b0` | 按钮 | 静态 | 返回首页 → `page main` |

### 标签说明

每个数字控件旁放静态文本标签:
- `n0` 旁: "DC 电机速度"
- `n1` 旁: "步进速度"
- `n2` 旁: "编码器计数"
- `n3` 旁: "左传感器"
- `n4` 旁: "右传感器"

### 状态颜色

在页面定时器中写逻辑(非 MCU 推送):
```
// 脱轨时 t1 变红
if(n3.val<30 && n4.val<30)  // 双传感器白色=脱轨
{
  t1.txt="脱轨! 已停车"
  t1.pco=RED
}
```

### 触摸事件

| 按钮 | 事件 | 执行代码 |
|------|------|----------|
| `b0` | 按下 | `page main` |

## 五、Page 3 — CAN 指令控制 `can_ctrl`

**定位:** 从 H753 下发运动指令到 G474 (CAN 0x201/0x202/0x203)。提供完整的 AGV/传送带/舵机控制面板。

**布局:** 三个控制区垂直排列。

### 4.1 AGV 底盘控制区

| 控件名 | 类型 | 用途 |
|--------|------|------|
| `t0` | 文本 | 区标题 "AGV 底盘控制" |
| `h0` | 滑块 | 速度设定 (0-255) |
| `n5` | 数字 | 速度值显示 (绑定 h0) |
| `r0` | 单选按钮组 | 方向: 前进/后退 (0/1) |
| `b4` | 按钮 | 发送 AGV 指令, 触摸代码: `printh 55 02 00`+h0.val低字节+`r0.val 00 ff ff ff` |

### 4.2 传送带控制区

| 控件名 | 类型 | 用途 |
|------|------|------|
| `t1` | 文本 | 区标题 "传送带控制" |
| `h1` | 滑块 | 转速设定 (0-255) |
| `n6` | 数字 | 转速值显示 (绑定 h1) |
| `r1` | 单选按钮组 | 方向: 正转/反转 (0/1) |
| `sw0` | 开关 | 启停 (0=停, 1=启) |
| `b5` | 按钮 | 发送传送带指令 |

### 4.3 舵机云台控制区

| 控件名 | 类型 | 用途 |
|------|------|------|
| `t2` | 文本 | 区标题 "舵机云台控制" |
| `h2` | 滑块 | 俯仰角度 (0-180) |
| `n7` | 数字 | 角度值显示 (绑定 h2) |
| `h3` | 滑块 | 偏航角度 (0-180) |
| `n8` | 数字 | 角度值显示 (绑定 h3) |
| `b6` | 按钮 | 发送舵机指令 |

### 触摸事件 (内部控制按键回到首页)

| 按钮 | 事件 | 执行代码 |
|------|------|----------|
| `b0` | 按下 | `page main` |

### 指令说明

CAN 指令由 MCU (app_h753.c) 处理，屏幕端按钮负责设置数值，由 MCU 读取后打包成 CAN 帧下发。也可以在使用 printh 指令直接从屏幕发送给 MCU 处理。此页控件由设计者自行根据 MCU 代码配合处理。

## 六、Page 4 — NFC 认证管理 `auth`

**定位:** 展示 NFC 身份认证状态、白名单管理、安全锁定状态。

### 控件清单

| 控件名 | 类型 | 用途 | MCU推送 |
|--------|------|------|---------|
| `t0` | 文本 | 页标题 "NFC 身份认证" | 静态 |
| `t1` | 文本 | 认证结果: 请刷卡/授权通过/未授权 | `Screen_SetText("t1","%s",msg)` |
| `t2` | 文本 | 卡片 UID: "A1B2C3D4" | `Screen_SetText("t2","UID:%s",uid)` |
| `t3` | 文本 | 白名单数量: "已授权 3/10 张" | `Screen_SetText("t3","已授权 %d/10",cnt)` |
| `t4` | 文本 | NFC 当前模式: IDLE/ADD/VERIFY | `Screen_SetText("t4","模式:%s",mode)` |
| `b0` | 按钮 | 返回首页 | 触摸 → `page main` |

### 状态颜色

TJC 上位机页面定时器:
```
// t1 文本根据内容变色
if(t1.txt=="授权通过") { t1.pco=GREEN }
if(t1.txt=="未授权")   { t1.pco=RED }
```

| 按钮 | 事件 | 执行代码 |
|------|------|----------|
| `b0` | 按下 | `page main` |

## 八、MCU ↔ 屏幕数据映射总表

### MCU → 屏幕 (数据推送)

| 数据来源 | 控件 | 页面 | 触发时机 |
|----------|------|------|----------|
| `status.dc_speed` | `n0` | main, motor | 每收到 0x301 帧 |
| `status.step_speed` | `n1` | main, motor | 每收到 0x301 帧 |
| `status.encoder_cnt` | `n2` | main, motor | 每收到 0x301 帧 |
| `status.sensor_left` | `n3` | motor | 每收到 0x301 帧 |
| `status.sensor_right` | `n4` | motor | 每收到 0x301 帧 |
| 分拣计数 | `n_sort` | main | 分拣完成一次 +1 |
| 系统运行状态 | `t_sensor` | main | 脱轨/超时/正常 |
| CAN 连接状态 | `t_conn` | main | 连接/断开 |
| NFC 认证结果 | `t_auth` | main | 已认证/未认证 |
| NFC 详情 | `t1,t2,t3,t4` | auth | NFC_Task 触发 |
| 系统模式 | `t_mode` | main | App_Init 初始化 |
| 硬件模块状态 | `t1~t6` | system | App_Init 初始化 |

### 屏幕 → MCU (触摸/控制事件)

当前通过 TJC 标准 `0x65` 触摸帧回传，MCU 端 `screen.c` 自动解析 page_id、ctrl_id、event_type 并通过回调通知应用层。

## 十、安全状态视觉规范

| 状态 | 颜色 | 页面表现 |
|------|------|----------|
| 正常运行 | 绿色/白色文字 | 所有数据显示正常 |
| 脱轨告警 | 红色背景闪烁 + 大字 "脱轨!" | main 页 t1 变红, motor 页 t1 变红 |
| 通信超时 | 黄色背景 + 大字 "通信中断" | main 页 t2 变黄 |
| NFC 未授权 | 红色文字 | auth 页 t1 红色 |
| NFC 已授权 | 绿色文字 + ✓ | auth 页 t1 绿色 |
| 电机锁定 | 灰色 | motor 页 t2 显示"已锁定" |
| 电机解锁 | 绿色 | motor 页 t2 显示"已解锁" |
